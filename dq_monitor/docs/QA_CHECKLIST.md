# QA Checklist — что проверять каждый день

Этот документ для **QA / Storyteller**. Здесь собрано всё, что вы делаете
на протяжении двух недель проекта. Роль QA — самая «горизонтальная»: вы
не пишете один большой модуль, а контролируете качество **всего**, плюс
готовите часть, которую увидит жюри (рекомендации, презентация, Q&A).

## Ваши главные обязанности

1. **Тесты на код команды** — pytest, по мере появления модулей
2. **Ручное тестирование** — каждый день прогон приложения «глазами жюри»
3. **Bug log** — ведение списка найденных проблем
4. **Бизнес-рекомендации** — это главная сцена защиты после Antifraud Demo
5. **Q&A документ** — заготовленные ответы на возможные вопросы жюри
6. **Документация** — README, инструкции, нарратив команды на защите

## Шаблон BUG_LOG.md

Создайте файл `docs/BUG_LOG.md` со следующим содержимым:

```markdown
# Bug Log

| Дата | Severity | Описание | Где | Кто чинит | Статус |
|------|----------|----------|-----|-----------|--------|
| 17.05 | High | После загрузки CSV метрика "транзакции" показывает 0 | streamlit_app.py | BE | open |
| 17.05 | Low | Опечатка в заголовке: "Антифорд" вместо "Антифрод" | страница 3 | FE | fixed |

## Severity
- **Critical** — приложение не запускается / падает в demo-сценарии
- **High** — основной сценарий работает, но с ошибкой результата
- **Medium** — мешает использовать, но обходится
- **Low** — косметика, опечатки, оформление
```

Каждый день добавляйте новые баги, помечайте закрытые.

## Что вы НЕ делаете

- **Не пишете бизнес-логику** в src/. Это работа DQ Analyst / Backend / Antifraud.
- **Не делаете дизайн страниц** — это Frontend / Viz.
- **Не ведёте борд задач** — это Team Lead.

Вы — это **«глаза заказчика» в команде** + **источник доверия к проекту**.
Хороший QA — это тот, при ком команда расслабленно говорит «мы готовы»,
потому что точно знает, что всё проверено.
